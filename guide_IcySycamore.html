Python 环境配置完全指南
团队标准工作流 · 兼容 Windows / macOS / Linux

🛠️ 团队技术栈 & 注意事项

核心栈： Python 3.12 + PyQt5/PySide6 + (ana)conda (或Miniconda) + VSCode
AI/CV 项目： PyTorch (CUDA), Open3D, trimesh, Fast3R 等。
⚠️ 易错点：
• 不要使用 Python 3.13+ (很多科学包未适配)
• 避免在 base 环境乱装包，务必创建虚拟环境
• 不要混用 conda 和 pip 安装同一核心包（如 numpy, torch）避免 DLL 冲突
• OpenMP 冲突（常见于 PyTorch + MKL）→ 设置环境变量 KMP_DUPLICATE_LIB_OK=TRUE

1. 什么是 Python 虚拟环境？为什么必须用？

虚拟环境是一个隔离的 Python 运行沙箱，每个项目拥有独立的包版本。重要性：

  • ✅ 避免包版本冲突（项目A需要 torch 1.5，项目B需要 torch 2.0）
  • ✅ 无需管理员权限，环境可随时删除重建
  • ✅ 与团队协作：提交 requirements.txt 即可一键复现环境
💡 核心理念：永远不在 base 环境工作！每次项目都创建专属环境。

2. pip, conda, Anaconda 三兄弟的关系

工具 定位 适用场景
pip Python 官方包管理器（PyPI） 纯 Python 包，轻量快速
conda 跨语言包管理器+环境管理 管理复杂二进制 (CUDA, OpenCV) + 环境
Anaconda 发行版 (conda + Python + 150+ 科学包) 新手，但体积大；追求轻量化可以使用Miniconda
⚠️ 项目成员请统一选择安装 conda ，用 conda 创建环境，用 pip/conda 安装 requirements.txt 中的包。

3. Anaconda 的三种操作界面（PowerShell / CMD / Navigator）

  • Anaconda PowerShell Prompt – 自动激活 base，推荐日常使用。
  • Anaconda CMD (prompt) – 适合脚本/CI。
  • Anaconda Navigator – 图形界面，适合新手或启动 Jupyter。
⚠️ 在 VSCode 中我们统一使用集成终端 + conda 初始化，见下一节

4. VSCode 中配置 Anaconda PowerShell 终端

❗ 常见错误：在 VSCode 终端输入 conda 提示 conda-script.py: error: argument COMMAND: invalid choice: ''
原因：VSCode 终端未加载 conda 初始化函数。

🔸 权宜之计，更健康且可能更有效的工作流见下一点（仅需做一次）
在 系统原生终端（不是 VSCode 内部）执行 conda init。

Windows (PowerShell)

# win+x 再按a
conda init powershell
conda init pwsh        # 如果你使用 PowerShell 7+

macOS / Linux (bash/zsh)

conda init
source ~/.bashrc      # 或重启终端

完成后关闭 VSCode 所有窗口，重新打开 → 新建终端即可正常使用 conda 命令。

💡 如果你不想让系统终端显示 (base) 且减少启动延迟，请看「9. conda init profile 取消延迟」章节。

5. conda 创建 & 管理虚拟环境

# 查看所有环境
conda env list

# 创建新环境 ( 指定 Python 版本，例如 3.12,其中-y是传递给命令行的参数，相当于对之后出现的y/n询问做确认 )
conda create -n 环境名称 python=3.12 -y

# 激活环境
conda activate 环境名称

# 退出当前环境
conda deactivate

# 删除环境（加 -y 自动确认）
conda env remove -n 环境名称 -y

# 克隆现有环境（作为备份）
conda create --name 新环境名 --clone 旧环境名
建议统一起名为Omni3D便于交流和管理（ 其实是直接复制命令 ）。

6. 安装依赖：pip vs conda 最佳实践

  • （可选）优先用 conda 安装 二进制复杂包：conda install -c conda-forge numpy pytorch opencv
  • 所有包统一用 pip 安装 requirements.txt，简单可靠。 
# 在 conda 环境内
conda activate Omni3D

pip install -r requirements.txt       # 安装项目依赖

# 如果上一条报错或出现问题，一般是deepspeed，
# open3d和vtk的问题
# windows系统不怎么适配deepseed，有方法可以但太麻烦，
# deepseed不是win用户必选项，当然你也可以选择linux或者WSL,
# 以下是一般性解决方案：
# 先在requirement.txt用#注释掉deepseed，然后用conda安装复杂包
conda install open3d pyqt5 vtk

# 然后注释掉open3d，因为已通过conda安装，
# 用pip安装requirements.txt
pip install -r requirements.txt

# 如果后续python报错缺少moudle ctype/qt库相关，这条用来解决ctypes依赖
conda install -c conda-forge libffi/pyqt5

示例：安装 PyTorch (以CUDA 12.1为例)

pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
⚠️ 自行查阅ai或访问nvida官网获取自己gpu版本及对应的安装命令/CUDA版本

从 requirements.txt 检查缺失包（不安装）

pip install -r requirements.txt --dry-run --ignore-installed

⚠️ 此条在不同win系统上实际执行情况存疑

7. 如何让 conda 也能使用 pip 编写的 requirements.txt？

❗ conda install --file requirements.txt 不支持 >=、~= 等版本约束符，仅接受包名或 ==version。

推荐方案：直接在 conda 环境里用 pip 安装 requirements.txt，这依然是最稳定且最简单的做法。

备选方案：编写 environment.yml 同时管理 conda 和 pip 依赖 

name: myenv
channels:
  - conda-forge
  - defaults
dependencies:
  - python=3.12
  - pip
  - pip:
      - -r requirements.txt

用下面命令一键创建环境：

conda env create -f environment.yml

8. conda init profile：为什么要 init？如何取消以减少启动延迟？

  • 为什么需要 conda init：它会将 conda 初始化代码写入 shell 配置文件（.bashrc, profile.ps1），使每次终端启动时自动激活 conda 函数。
  • 副作用：终端启动变慢（1200-2000ms），并且显示 (base) 前缀。
  • So,如何取消（恢复干净的终端）？

Windows (PowerShell)

conda init --reverse powershell

macOS / Linux

conda init --reverse

💡 更好的折中：只在 VSCode 内部启用 conda，而不污染全局终端 → 见章节4的“仅影响 VSCode 终端的配置文件”。

9. 常见包冲突解决：OpenMP 运行时冲突以及 DLL 丢失

🔸 典型错误：OMP: Error #15: Initializing libomp.dll, but found libiomp5md.dll already initialized.

原因：同时加载了 Intel MKL 的 OpenMP 与 LLVM 的 OpenMP。

✔ 快速解决（推荐）：运行脚本前设置环境变量。

# Windows PowerShell 临时设置
$env:KMP_DUPLICATE_LIB_OK="TRUE"

# macOS / Linux
export KMP_DUPLICATE_LIB_OK=TRUE

或在 Python 代码开头加入：

import os
os.environ["KMP_DUPLICATE_LIB_OK"] = "TRUE"

🔸 （据说的）根本清理方案（能跑起来就行，没必要）

# 方法 A：保留 Intel，移除 LLVM
conda remove llvm-openmp -y
conda install intel-openmp -c conda-forge -y

# 方法 B：若冲突依旧，手动重命名 libiomp5md.dll 备份
# 路径： <环境目录>\Library\bin\libiomp5md.dll

🔸 另一个常见问题：ImportError: DLL load failed while importing _ctypes

原因：缺失 ffi.dll（conda 环境更新导致的残留）。手动从 conda 缓存恢复： 

# 找到 libffi 缓存目录
cd D:\anaconda3\pkgs
dir libffi-*

# 复制 ffi.dll 到当前环境 (例：环境名为 Omni3D)
copy libffi-3.4.4-hd77b12b_1\Library\bin\ffi.dll D:\anaconda3\envs\Omni3D\Library\bin\
